# 问题路由对照表

## 目的

这份文档用于帮助项目实际操作者快速判断：

- 当前问题应该先问谁
- 哪类问题由哪个角色负责
- 遇到分歧时由谁拍板

## 使用原则

- 业务方案、页面定义、验收口径，优先问产品经理 P
- 技术设计、实现路径、接口数据，优先问技术负责人 T
- 方向边界、优先级和冲突拍板，优先问总负责人

## 对照表

| 问题类型 | 优先问谁 | 原因 |
| --- | --- | --- |
| 这个需求要不要做 | 总负责人 | 涉及方向和范围判断 |
| 这件事要不要进 V1 | 总负责人 | 涉及版本边界 |
| P 和 T 说法不一致怎么办 | 总负责人 | 需要统一拍板 |
| 现在该先推进产品还是技术 | 总负责人 | 涉及优先级排序 |
| 这个方案是否偏离目标 | 总负责人 | 涉及产品一致性 |
| 业务流程怎么设计 | 产品经理 P | 属于产品方案 |
| 页面怎么定义 | 产品经理 P | 属于产品交互 |
| 报告该展示什么 | 产品经理 P | 属于产品输出 |
| 验收标准怎么写 | 产品经理 P | 属于产品口径 |
| 某个业务规则怎么表达成产品要求 | 产品经理 P | 属于需求整理 |
| 接口怎么设计 | 技术负责人 T | 属于技术设计 |
| 数据结构怎么定 | 技术负责人 T | 属于技术设计 |
| 状态流转怎么做 | 技术负责人 T | 属于技术实现方案 |
| 文件解析怎么落地 | 技术负责人 T | 属于技术实现 |
| AI 审查链路怎么编排 | 技术负责人 T | 属于技术架构 |
| 模块怎么拆 | 技术负责人 T | 属于技术方案 |
| 开发顺序怎么排 | 技术负责人 T | 属于技术执行方案 |

## 一句话记忆

- 问 P：产品要什么
- 问 T：技术怎么做
- 问总负责人：这事该不该做、先做什么、谁说了算

## 分歧处理

当出现以下情况时，应交给总负责人统一收口：

- P 与 T 对边界理解不一致
- 技术方案明显影响产品范围
- 当前优先级存在争议
- 是否进入下一阶段无法自行判断

## 当前结论

这张对照表的目标，不是限制沟通，而是减少角色混用和重复讨论，提高协作效率。
