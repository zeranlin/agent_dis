# 收件箱归档说明

## 目的

这个目录用于存放已经处理完成的历史消息，避免 `inbox-owner.md`、`inbox-p.md`、`inbox-t.md` 无限增长，减少线程读取时的 token 消耗。

## 使用原则

- `inbox-*.md` 只保留待处理和最近仍需要参考的消息
- 已经处理完成的历史消息，定期移入归档文件
- 归档文件按角色拆分，避免单文件过大

## 建议归档文件

- `owner-history.md`
- `p-history.md`
- `t-history.md`

## 当前结论

收件箱负责当前沟通，归档目录负责历史保留。这样可以同时保证可追溯和低 token 使用。
