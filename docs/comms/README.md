# 仓库内通信机制

## 目的

这个目录用于支持总负责人、产品经理 P、技术负责人 T 之间通过仓库文件和 git 进行异步通信，减少手工复制聊天内容。

## 设计目标

- 让问题和回复留存在仓库中
- 让不同线程通过拉取最新代码读取消息
- 让关键沟通可追溯
- 减少跨线程重复转述

## 文件说明

- `inbox-owner.md`：发给总负责人的问题、请示和待拍板事项
- `inbox-p.md`：发给产品经理 P 的问题、澄清和需求
- `inbox-t.md`：发给技术负责人 T 的问题、安排和反馈
- `template.md`：消息模板

## 使用方式

推荐按以下方式使用：

1. 某个线程把问题写入对应 inbox
2. 提交并推送到远端
3. 另一个线程拉取最新代码
4. 读取对应 inbox 并处理
5. 将回复追加写入对应 inbox，或写回正式文档与任务状态

## 使用原则

- 正式问题优先写入 inbox，不只留在聊天里
- 每条消息都要写清主题、背景、问题和建议动作
- 已经形成正式结论后，应同步写入正式文档，而不是只留在 inbox
- `docs/tasks/current.md` 继续作为总状态板，`docs/comms/` 只负责消息往来

## 当前结论

这套机制的目标不是替代正式文档，而是让线程间沟通先有一个稳定、可追溯的落点。
