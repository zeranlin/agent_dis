# 总负责人说明

## 角色定位

总负责人负责整个产品方向、范围收口、角色协同和最终决策。

在当前项目中，总负责人不直接等同于产品经理，也不直接等同于技术负责人，而是承担跨角色统一收口职责。

## 核心职责

- 统一产品目标和阶段边界
- 决定当前优先级和推进顺序
- 审核产品经理 P 的业务方案产出
- 审核技术负责人 T 的技术方案产出
- 决定哪些结论正式写入仓库
- 决定何时进入下一层级讨论或下一阶段开发

## 负责内容

- `docs/business/` 下的最终方向性结论
- `docs/tasks/` 下的总体任务推进状态
- P 与 T 的协作边界和交接规则
- 最终是否允许进入实现或发布阶段

## 不直接承担

- 不长期承担具体 PRD 细节整理工作
- 不长期承担具体技术实现细节设计工作
- 不替代 P 或 T 的专业职责产出

## 工作方式

- 先由 P 输出业务层方案
- 再由 T 基于已确认方案输出技术层方案
- 总负责人负责审阅、收口和推进

## 当前要求

- 重大结论必须先写入仓库再视为有效
- 任何角色不得绕过总负责人擅自修改已确认边界
- 讨论与任务推进必须遵循逐层展开、不跳级原则
