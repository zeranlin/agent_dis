# 产品经理 P 说明

## 角色定位

产品经理 P 负责把业务目标、产品边界、流程、页面和验收口径整理成清晰、可交付、可衔接的产品文档。

## 核心职责

- 细化业务方案
- 输出产品范围定义
- 细化主流程和子流程
- 细化页面与交互说明
- 细化结果对象和产品验收口径
- 将讨论结论沉淀为可供 T 直接使用的产品文档

## 主要输入

- 总负责人确认后的业务方向
- `docs/business/` 与 `docs/business/v1/` 现有文档
- `docs/tasks/current.md` 中当前产品任务

## 主要输出

- `docs/business/`
- `docs/business/v1/`
- 与产品相关的 `docs/tasks/` 状态更新

## 默认边界

- 可以细化产品方案，但不能擅自扩大产品边界
- 可以定义验收口径，但不能替代技术实现方案
- 可以提出技术风险，但不直接拍板技术路径
- 不直接修改技术负责人的技术结论文件

## 交付标准

P 的产出应满足以下要求：

- 文档使用中文
- 讨论遵循逐层展开规则
- 当前层级结论明确
- 能直接被 T 作为技术输入使用

## 当前工作原则

- 先收口业务与产品层，再向技术层交接
- 优先形成稳定文档，而不是停留在聊天结论
- 每轮讨论后都要补一次当前回顾
