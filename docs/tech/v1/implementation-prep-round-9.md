# V1 实现准备第九轮（首版）

## 文档目的

这份文档用于承接结果页最小实现成果，继续收口结果页的信息组织、快速操作区和更完整的页面交互反馈。

## 1. 产品追溯

本轮实现直接追溯以下已确认文档：

- `docs/status/summary.md`
- `docs/tasks/t-now.md`
- `docs/tasks/current.md`
- `docs/business/v1/page-and-result-spec.md`
- `docs/tech/v1/api-spec.md`
- `docs/tech/v1/implementation-prep-round-8.md`

若后续发现结果页细化方向与产品边界冲突，应先回提总负责人，不直接扩成复杂前端项目。

## 2. 本轮目标

本轮只聚焦三件事：

1. 优化结果页信息组织
2. 增加更清晰的快速操作与入口说明
3. 继续补全页面级交互反馈

## 3. 本轮范围

### 3.1 本轮要做

1. 对完成态页面做更清晰的区块拆分
2. 增加快速操作区
3. 补充页面地址、状态接口、结果接口说明
4. 对审核中与失败态保留更明确的下一步动作提示
5. 补齐对应最小测试

### 3.2 本轮不做

- 富文本样式系统
- 前端框架改造
- 风险筛选与排序交互
- 权限与登录体系

## 4. 当前实现设计

### 4.1 完成态信息组织

当前完成态页面拆为两列：

1. 主内容列
2. 辅助操作列

主内容列包括：

- 风险统计
- 重点风险摘要
- 最终结论
- 审查报告

辅助操作列包括：

- 下载入口
- 页面入口说明
- 状态接口入口
- 结果接口入口
- 查看建议

### 4.2 快速操作区

当前新增“快速操作”区块，目标是让结果页在极简条件下也具备明确操作路径。

当前展示内容包括：

- 下载最终结论
- 下载审查报告
- 当前页面地址
- 状态接口地址
- 结果接口地址

### 4.3 页面级交互补全

当前三类状态进一步明确如下：

- `completed`：展示结果内容、下载入口和查看建议
- `reviewing`：展示等待说明，并提示可轮询状态接口与结果接口
- `failed`：展示错误码，并提示可先检查状态接口和重新提交文件

## 5. 最小测试

本轮至少覆盖以下验证：

1. 完成态页面展示快速操作区
2. 完成态页面展示页面/状态/结果入口说明
3. 结果接口返回页面接入所需入口字段

## 6. 当前结论

实现准备第九轮的目标不是继续扩 UI 复杂度，而是把现有结果页做得更清晰、更可用。

完成这一轮后，当前结果页已经不只是“能打开”，而是具备：

- 更清晰的信息组织
- 更明确的操作入口
- 更完整的页面级反馈

后续可继续进入页面布局细节优化或更细的交互补全。
