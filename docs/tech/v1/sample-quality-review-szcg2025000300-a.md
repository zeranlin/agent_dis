## 问题归类

## 一、提示词问题

当前最突出。

表现：

- 模型倾向先输出“模糊但容易解释”的泛化风险
- 对“品牌/原厂/厂家认证/评分量化”这类高价值招采风险优先级不够
- 对同一条款会输出多个近义风险，而不是先收敛成一个更强风险

下一轮建议：

1. 在提示词中明确“优先识别高价值公平竞争风险，再识别模糊性风险”
2. 明确要求先看 `R1` / `R3` / `R5` / `R9` 这类高价值规则
3. 明确要求同一条款相近风险优先合并，不要重复输出近义结论
4. 增加少量正反例，特别是“同品牌”“原厂保修”“厂家认证讲师”这类案例

## 二、规则包问题

当前问题也很明显。

表现：

- `R2` 被拿去承接“最低报价限制”
- `R12` 被拿去承接“第三方检测费用承担不公平”
- 规则标题和真实业务语义存在偏移

下一轮建议：

1. 为现有规则补“命中定义”“不该命中的反例”“典型表述”
2. 对 `R2`、`R12` 这类容易漂移的规则补更严格边界
3. 若总负责人允许，再评估是否需要把“价格竞争限制”“合同责任失衡”单列为后续候选规则，但不建议在当前轮直接扩范围落实现

## 三、结构切分问题

当前存在，但不是第一优先级。

表现：

- `location_label` 过长，整段大条款直接塞进定位标签
- 原文与“商务要求偏离表”存在重复区域，后续容易引发重复命中
- 培训方案类长条款承载多个子要求，单条过长，容易让模型在一个大段内输出多个相近风险

下一轮建议：

1. 对超长条款增加更细粒度子切分
2. 对“偏离表 / 响应表 / 正文重复块”做去重或降权
3. 缩短 `location_label`，把完整原文留给正文字段，不要让定位字段承载过多文本

## 四、结果归并与报告问题

当前是第二优先级问题。

表现：

- 同一条款同一规则被重复提示
- 总体结论直接按当前风险级别计数，容易被“挂错高风险标题”带偏
- 报告里缺少“重复合并后风险组”的视角

下一轮建议：

1. 增加“同条款 + 同规则 + 近义说明”的轻量归并
2. 总体结论优先面向“风险组”而不是“原始命中条数”
3. 在结果页或报告中显式标出“需人工确认”与“规则归类置信不足”信号

## 下一轮精度优化建议整包

本轮建议下一轮按以下顺序推进，不跨主题混包。

## 第一优先级：提示词与规则包联合收口

目标：

- 先提升高价值风险召回
- 压低规则语义漂移

建议动作：

1. 重写审查提示词中的优先级说明
2. 为 `R1` / `R3` / `R5` / `R9` / `R12` 增加命中说明与正反例
3. 对“同品牌”“原厂保修”“厂家认证讲师”“执业医师证”“评分量化”加入显式关注项

## 第二优先级：结果归并轻收口

目标：

- 先压掉明显重复
- 避免总体结论被原始命中数带偏

建议动作：

1. 对同条款同规则重复命中做轻量归并
2. 报告按风险组展示，不直接平铺全部原始命中
3. 总体结论按风险组和最高风险组级别生成

## 第三优先级：长条款细切分与重复块降权

目标：

- 为下一轮更细规则命中提供更干净输入

建议动作：

1. 对超长商务条款做子切分
2. 对偏离表和正文重复块做识别
3. 降低重复块在审查输入中的权重
