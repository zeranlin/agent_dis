# `SZCG2025000300-A` 人工差异导向的规则补强方案

## 文档目的

本文件用于把 `SZCG2025000300-A` 的人工审核结果与当前系统审查结果之间的主要差异，收敛成下一轮最值得补强的 5 组规则能力。

本轮目标不是继续做结果展示微调，而是把“人工已明确命中、系统仍大量漏报”的部分，整理成可以直接进入开发的规则专题包。

## 一、当前差异的核心判断

基于人工审核表与当前系统对 `SZCG2025000300-A` 的阶段结果对比，当前差异有两个明显特征：

1. 系统已经能较稳定识别一部分高价值风险

- 同品牌约束
- 检测费用承担与合同责任失衡
- 原厂保修与培训资源偏严

2. 系统对人工审核表中的大量规则型风险仍覆盖不足

主要缺口集中在：

- 资格条件违法限定
- 评分因素与评分方法违法
- 检测机构与检测标准合规
- 技术参数区间与冲突检查
- 价格分规则合法性

因此，下一轮主线不应继续优先做报告层润色，而应优先补齐上述 5 组规则能力。

## 二、补强原则

### 2.1 先补规则型、确定性强的问题

优先补那些：

- 法规口径明确
- 文本模式相对稳定
- 可以先用确定性识别吃掉大头

### 2.2 先做“能覆盖人工埋点”的版本

本轮目标不是一步做到完整规则库，而是先让系统对 `SZCG2025000300-A` 人工审核表中的主要埋点具备基础覆盖能力。

### 2.3 仍坚持三层识别结构

每个专题包都按以下顺序设计：

1. 确定性识别
2. 弱语义规则识别
3. 不确定区交给 `LLM` 补判

### 2.4 不把所有问题继续压到 `R12`

当前 `R12` 已承担过多合同与程序兜底责任。

本轮补强时应尽量把：

- 资格问题
- 评分问题
- 检测标准问题
- 参数区间问题
- 价格分问题

从“兜底语义”中拆出来，进入更接近业务语义的专门规则。

## 三、5 组规则能力专题包

## 3.1 资格条件违法限定

### 目标

识别不应作为准入资格的规模、年限、区域、行业、信用、证书等限制。

### 对应人工埋点

对应 `xlsx` 中以下问题：

1. 全国科技型中小企业
2. 高新技术企业证书
3. 纳税信用 `A` 级证明
4. 成立满 `5` 年
5. 深圳市医疗器械行业同类项目业绩不少于 `2` 个

### 当前系统短板

当前系统对这类问题没有稳定形成有效输出，说明：

- 资格模块规则边界仍偏粗
- `R3` 主要覆盖了“资格偏严”，但尚未细到“哪些条件本身就不宜作为资格”

### 建议拆分的能力点

1. 规模与财务能力不得作为资格条件
2. 成立年限、经营年限不得作为资格条件
3. 特定区域、特定行业、特定数量业绩不得作为资格条件
4. 与项目不直接相关的证书、认证、信用等级不得作为资格条件

### 建议实现方式

确定性识别：

- `成立满X年`
- `纳税信用A级`
- `高新技术企业`
- `科技型中小企业`
- `深圳市.*业绩`
- `同类项目业绩不少于`

弱语义规则识别：

- 标题位于 `资格要求 / 投标人资格 / 资格条件`
- 正文出现 `投标人须 / 应具备 / 提供证明 / 提供证书`
- 并同时出现 `年限 / 信用 / 高新 / 科技型 / 区域 / 行业 / 业绩数量`

`LLM` 补判：

- 判断某项证书是否与项目标的直接相关
- 判断某项业绩限制是否具有明显区域或行业排他性

### 产出建议

建议优先在现有规则包上补强：

- `R3`：资格条件过高检查
- `R4`：业绩要求定向检查

必要时后续再讨论是否拆出更细规则，不在本轮先扩编号。

## 3.2 评分因素 / 评分方法违法

### 目标

识别不应作为评审因素的规模财务指标、成立年限、无关证书，以及主观性过强或与项目无关的评分项。

### 对应人工埋点

对应 `xlsx` 中以下问题：

1. 资产总额达到 `5000` 万以上得分
2. 从业人员超过 `50 / 100` 人得分
3. 近三年年均纳税额达到 `200` 万以上得分
4. 成立时间满 `5 / 10 / 15` 年得分
5. `人力资源测评师`
6. 非金属矿采矿许可证
7. 操作体验、界面友好美观、座椅设计美观

### 当前系统短板

当前评分问题只有少量 `R9` 能命中，且更多集中在量化不足，尚未覆盖：

- 不得设为评审因素的规模条件
- 与采购标的不相关的证书
- 明显主观描述打分

### 建议拆分的能力点

1. 规模财务指标不得作为评审因素
2. 成立年限不得作为评审因素
3. 无关证书或已取消资质不得作为评审因素
4. 主观描述不得直接作为评审依据

### 建议实现方式

确定性识别：

- `资产总额.*得[分分值]`
- `从业人员.*得[分分值]`
- `纳税额.*得[分分值]`
- `成立时间.*得[分分值]`
- `许可证.*得[分分值]`

弱语义规则识别：

- 标题位于 `评分标准 / 评分表 / 商务评分 / 技术评分`
- 文本同时出现 `得X分 / 满分 / 评分项`
- 并出现 `资产总额 / 从业人员 / 纳税额 / 成立时间 / 证书 / 美观 / 体验`

`LLM` 补判：

- 判断某证书或资质与采购标的是否相关
- 判断某主观描述是否缺乏可量化评价标准

### 产出建议

建议优先补强：

- `R9`：评分项未量化检查
- `R11`：重复加码检查

并新增一组“评分因素合法性”识别逻辑，先不急着扩规则编号。

## 3.3 检测机构与检测标准合规

### 目标

识别指定非规定检测机构、使用无效或不存在标准、检测责任设置不当等问题。

### 对应人工埋点

对应 `xlsx` 中以下问题：

1. 深圳市医疗器械检测中心出具检测报告
2. `GB/T 99999-2024《医疗机器人通用技术规范》`
3. 第三方检测费用由中标人承担，无论检测结果是否合格

### 当前系统短板

当前只有“检测费用承担”能较稳定进入结果，另外两类问题基本缺位：

- 指定检测机构
- 使用不存在标准

### 建议拆分的能力点

1. 非行政机关明确要求时，不得限定特定检测机构
2. 不得引用无效、过期或不存在的标准
3. 检测费用与检测结果责任承担不得失衡

### 建议实现方式

确定性识别：

- `须提供.*检测中心出具`
- `指定.*检测机构`
- `GB/T|GB|YY|ISO` 等标准号模式提取

弱语义规则识别：

- 标题位于 `检测要求 / 验收要求 / 技术要求 / 商务要求`
- 正文同时出现 `检测报告 / 出具 / 标准 / 规范 / 第三方检测`

`LLM` 补判：

- 判断标准号是否疑似不存在或与采购标的不匹配
- 判断指定检测机构是否存在法规例外情形

### 产出建议

建议：

- 保留 `R12` 承接“检测费用与责任失衡”
- 新增“检测机构与标准合规”专题逻辑
- 后续可再讨论是否演进为独立规则

## 3.4 技术参数区间与冲突检查

### 目标

识别技术参数只写绝对值、区间描述缺失、同一指标前后冲突等问题。

### 对应人工埋点

对应 `xlsx` 中以下问题：

1. `产品响应时间：100ms`
2. `手术机械臂精度：0.1-0.5mm`
3. `位置A` 不允许正偏离，`位置B` 允许 `±10%` 偏差

### 当前系统短板

当前系统在技术模块更擅长识别品牌指向，不擅长识别：

- 参数区间描述是否合法
- 同一参数前后要求是否冲突

### 建议拆分的能力点

1. 绝对值参数是否缺少合理区间说明
2. 区间表述是否自相矛盾
3. 同一技术指标是否存在互相冲突的偏离要求

### 建议实现方式

确定性识别：

- 抽取 `≤ / ≥ / = / ± / - / ～`
- 抽取数值 + 单位
- 同名参数多次出现时做冲突比对

弱语义规则识别：

- 标题位于 `技术要求 / 参数表 / 偏离表`
- 同一参数名称在相邻片段中出现多个不同约束

`LLM` 补判：

- 判断某个绝对值参数是否需要区间化表达
- 判断多个约束是否构成实际冲突

### 产出建议

建议优先作为 `R6`、`R7` 的技术子能力扩展：

- `R6` 更偏“参数组合与参数约束”
- `R7` 更偏“技术要求表述不清或无法执行”

本轮先做专题逻辑，不先扩编号。

## 3.5 价格分规则合法性

### 目标

识别综合评分法下未采用低价优先法、价格分计算公式错误等问题。

### 对应人工埋点

对应 `xlsx` 中以下问题：

1. 价格分采用中间价优先法，而不是低价优先法

### 当前系统短板

这类问题规则口径非常明确，但当前系统仍未稳定命中，说明：

- 评分模块当前更偏“主观量化不足”
- 对价格分方法合法性的专门识别还没建立起来

### 建议实现方式

确定性识别：

- `价格分计算方法`
- `评标基准价`
- `去掉最高价和最低价`
- `平均价`
- `中间价优先`
- `低价优先`

弱语义规则识别：

- 标题位于 `价格分 / 评分办法 / 评标方法`
- 文本出现公式说明、评标基准价、报价分计算

`LLM` 补判：

- 判断当前价格分公式是否等价于低价优先法

### 产出建议

建议先补在评分模块专题内，作为：

- `R9` 的价格分方法子场景

如果后续真实样例持续出现，再讨论是否拆成独立规则。

## 四、开发优先级

基于法规确定性、文本模式稳定性和对人工埋点覆盖价值，建议优先级如下：

1. 资格条件违法限定
2. 评分因素 / 评分方法违法
3. 价格分规则合法性
4. 检测机构与检测标准合规
5. 技术参数区间与冲突检查

排序原因：

- 前 3 组更接近法规明确、模式稳定、容易做出效果
- 后 2 组更依赖结构化抽取、规则组合或语义补判

## 五、建议的数据与执行改动

为支持这 5 组能力，建议补以下能力，但仍保持 `V1` 极简边界：

### 5.1 解析层

建议补：

- 资格条件片段的更细分类
- 评分项与价格分规则的独立标签
- 技术参数名称、数值、单位的最小抽取
- 标准号模式抽取

### 5.2 候选筛选层

建议补：

- 资格类规则优先看资格模块与资格审查表
- 评分类规则优先看评分表、评分项、价格分规则
- 检测类规则优先看技术要求、验收条款、商务要求
- 参数区间类规则优先看参数表、偏离表

### 5.3 审查执行层

建议补：

- 每组专题的确定性命中优先
- 再把不确定片段送给 `LLM`
- 不再把大量本可代码判定的问题都交给 `LLM`

## 六、给技术负责人的实施建议

建议按“两轮两包”推进：

### 第一轮

先做：

1. 资格条件违法限定
2. 评分因素 / 评分方法违法
3. 价格分规则合法性

目标：

- 先覆盖人工审核表中的大头漏报
- 先让 `300-A` 的人工埋点有明显提升

### 第二轮

再做：

1. 检测机构与检测标准合规
2. 技术参数区间与冲突检查

目标：

- 补齐更依赖结构与规则组合的能力

## 七、本轮结论

本轮正式结论如下：

1. `SZCG2025000300-A` 的人工审核结果已经把当前系统的主要漏报方向暴露得很清楚
2. 当前最值得优先补强的不是页面、报告或泛泛规则扩张，而是这 5 组能力专题
3. 这 5 组能力覆盖了当前人工审核表的大部分漏报点
4. 若下一轮继续推进，应优先从“资格 + 评分 + 价格分”三组开始，再补“检测 + 参数区间”
