# 结果归并轻收口首包回提

## 文档目的

本文件用于记录“结果归并轻收口”首包的实现范围、真实样例联调结果和当前正式判断。

本轮严格限定在以下范围内：

1. 对同条款 + 同规则 + 近义风险说明做轻量归并
2. 压降明显重复命中，避免风险数量膨胀
3. 让总体结论优先基于风险组而不是原始命中条数
4. 让结果页与 Markdown 报告更偏向风险组视角
5. 使用 `SZCG2025000300-A` 重新联调验证效果

本轮不处理：

1. 页面扩展
2. 流程扩展
3. 长条款细切分与重复块降权
4. 提示词与规则包继续扩张

## 本轮实现

### 1. 结果层新增轻量风险组归并

已在 `app/result_presenter.py` 新增结果层归并能力。

当前归并策略如下：

1. 同 `clause_id + rule_id` 直接归并
2. 同 `location_label + rule_id` 直接归并
3. 同章节下、同规则且风险说明高度相似的结果轻量归并

当前归并输出会保留：

1. 代表风险标题
2. 风险级别
3. 命中位置
4. 合并后的风险说明
5. 合并后的审查说明
6. 合并命中数

### 2. 总体结论改为基于风险组

已更新 `app/result_aggregator.py`。

当前总体结论不再基于原始 `risk_item` 条数，而是基于归并后的风险组数量生成。

对应变化包括：

1. `overall_conclusion` 改为“归并后存在 N 组高风险问题”
2. `conclusion_markdown` 中的风险统计明确说明为归并后的风险组数量
3. `report_markdown` 改为输出“风险组明细”

### 3. 结果页与结果接口改为风险组视角

已更新：

1. `app/upload_service.py`
2. `app/result_page.py`

当前变化包括：

1. `top_risks` 改为取归并后的风险组摘要
2. 结果页统计卡片改为“高风险组 / 中风险组 / 低风险组”
3. 结果页重点摘要改为“优先关注的风险组”
4. 风险摘要中补充“归并命中数”

### 4. 自动化验证补充

已补以下测试：

1. 同 `clause_id + rule_id` 的重复命中会归并
2. 同 `location_label + rule_id` 的重复命中会归并
3. 报告输出改为风险组明细
4. 结果页与结果接口输出风险组视角字段

## 前后差异

### 1. 总体结论口径已切换

上一轮结论口径仍容易被原始命中条数直接带偏。

本轮改为：

- `本文件归并后存在 13 组高风险问题，建议优先复核资格条件、评分规则或定向限制条款。`

这意味着结论层已经从“命中了多少条”切到“可复核风险组有多少组”。

### 2. 结果页与 Markdown 展示口径已切换

本轮已不再强调平铺原始命中，而是：

1. 页面展示风险组统计
2. 页面重点区展示风险组摘要
3. 报告正文展示风险组明细
4. 每个风险组补充归并命中数

### 3. 数量压降已发生，但收益有限

本轮从结果层轻量归并后：

1. 原始命中总数从 `18` 压到 `17` 组
2. 中风险从 `5` 压到 `4` 组

说明：

1. 最明显的同条款近义重复已能被结果层压掉
2. 但当前样例中的大量重复并不全是“同条款同规则”级别
3. 很多重复更接近长段条款拆分、相邻片段重复或正文/偏离表平行区域重复

因此，本轮收益是`结论口径和展示口径已收口`，但`数量压降幅度仍有限`。

## 是否存在阻塞性交付缺口

当前判断：`不存在新的阻塞性交付缺口`

原因：

1. 本轮改动未破坏上传、解析、审查执行、结果查看和下载链路
2. 结果页、结果接口和 Markdown 报告口径已完成一致性切换
3. `make check` 已通过

但需要明确：

- 当前仍存在质量层面的剩余问题，不影响当前最小交付闭环可用性，但会影响结果数量与人工阅读成本

## 是否需要进入第三优先级主题

当前判断：`建议进入第三优先级主题`

原因：

1. 结果层轻量归并已经把“显性重复”收口到位
2. 当前剩余重复更多来自长条款细切分不足与重复块放大
3. 继续只在结果层加码，边际收益会明显下降

因此，若总负责人继续放行，下一轮应进入：

- `长条款细切分与重复块降权`
