# `SZCG2025000300-A` 人工差异导向规则补强第一轮实现

## 文档目的

本文件记录 `SZCG2025000300-A` 人工差异导向规则补强第一轮的正式实现结果。

本轮只覆盖以下 `3` 组能力：

1. 资格条件违法限定
2. 评分因素 / 评分方法违法
3. 价格分规则合法性

本轮目标不是继续扩页面、扩解析大主题或重写规则体系，而是在现有 `V1` 主链路里补一层可落地的规则型识别能力，让人工审核表中的主要漏报点先出现一轮明显覆盖提升。

## 实现范围

本轮继续保留以下既有能力：

1. 文本提取
2. 基础切块
3. 业务单元识别
4. 候选送审筛选
5. `LLM` 收缩执行
6. 结果汇总与报告输出

本轮新增内容只落在 [review_executor.py](../../app/review_executor.py)：

1. 在现有 `LLM` 审查前增加“确定性识别 + 弱语义规则”的预命中层
2. 预命中结果与 `LLM` 返回结果在同一批次内并流
3. 同条款同规则按 `clause_id + rule_code` 去重，避免一条规则重复落库

## 本轮实现内容

## 一、资格条件违法限定

本轮补了 `R3 / R4` 的首轮确定性识别。

### 当前已覆盖的典型表达

1. `高新技术企业`
2. `科技型中小企业`
3. `纳税信用 A 级`
4. `成立满 X 年`
5. `深圳市 ... 同类项目业绩不少于 X 个`
6. `同类项目业绩不少于`

### 当前落库口径

1. `R3`
   - `资格条件设置无关资质或信用要求，存在违法限定风险`
   - `资格条件设置成立年限要求，存在违法限定风险`
2. `R4`
   - `资格业绩要求存在区域或数量定向风险`

### 当前判断

本轮已经把“资格偏严”往前推进到“哪些资格条件本身就不宜作为准入门槛”的首轮可用版本。

## 二、评分因素 / 评分方法违法

本轮补了 `R9` 的首轮确定性识别，不再只覆盖“优良中差式主观评分”。

### 当前已覆盖的典型表达

1. `资产总额达到 ... 得分`
2. `从业人员超过 ... 得分`
3. `纳税额达到 ... 得分`
4. `成立时间满 ... 得分`
5. `许可证 / 证书 / 认证 / 测评师 ... 得分`
6. `美观 / 体验 / 友好 / 酌情 / 综合评价`

### 当前落库口径

统一继续落在 `R9`，但标题已能区分为：

1. `评分因素设置不当，存在以规模财务指标打分风险`
2. `评分因素设置不当，存在以成立年限打分风险`
3. `评分因素设置不当，存在以无关证书资质打分风险`
4. `评分标准量化不足，自由裁量空间较大`

### 当前判断

本轮已经把“评分问题”从单一量化不足，推进到“评分因素合法性 + 评分方法违法”的首轮可用版本。

## 三、价格分规则合法性

本轮把价格分子场景并入 `R9` 的首轮识别逻辑。

### 当前已覆盖的典型表达

1. `评标基准价`
2. `基准价`
3. `平均价`
4. `算术平均价`
5. `最接近基准价者得满分`

### 当前落库口径

统一继续落在 `R9`：

- `价格分计算方法疑似不符合低价优先原则`

### 当前判断

本轮已经把“价格分规则合法性”从空白状态推进到首轮可识别版本。

## 技术方案落点

本轮没有新建平行目录，也没有引入新的外部依赖。

当前技术落点如下：

1. 在审查执行阶段增加 `_build_heuristic_findings(...)`
2. 按业务模块和业务单元限定触发范围，避免泛化误报
3. 预命中结果与 `LLM` 结果统一走现有风险落库链路
4. 批次内按 `clause_id + rule_code` 去重，减少重复命中

## 验证结果

本轮已补并通过以下自动化验证：

1. 资格条件违法限定识别
2. 评分因素 / 评分方法违法识别
3. 价格分规则合法性识别
4. 无 `LLM` 命中时，预命中结果仍可进入主链路落库
5. 既有 `R9` / `R12` 纠偏与结果展示回归未破坏

## 本轮结论

本轮正式结论如下：

1. `SZCG2025000300-A` 人工差异导向规则补强第一轮已完成首包实现
2. 资格条件违法限定、评分因素 / 评分方法违法、价格分规则合法性这 `3` 组能力已进入现有主链路
3. 当前实现仍保持 `V1` 极简边界，没有扩页面、流程或平行实现目录
4. 当前更适合进入下一轮能力补强，而不是回头继续做页面或报告微调

## 剩余缺口

本轮仍保留以下缺口，但不在本轮继续扩：

1. 检测机构与检测标准合规
2. 技术参数区间与冲突检查
3. 本轮新增规则型命中尚未做真实 `LLM` 样例复跑核对

建议按以下顺序推进：

1. 先以 `SZCG2025000300-A` 重新做一轮结果核对，确认人工埋点覆盖提升是否兑现
2. 若本轮提升已明显兑现，再进入剩余 `2` 组规则补强专题
3. 若仍存在明显漏报，只在当前 `3` 组主题内继续补最小规则，不回头扩新主题
