# V1 实现准备第二十八轮（首版）

## 文档目的

这份文档用于承接“直接进入真实样例联调与验收走查”的最新指令，记录 5 类样例文件的联调结果，并按“通过 / 不通过”给出本轮正式判断。

本轮不再继续补实现，除非联调中发现新的阻塞性交付缺口。

## 1. 联调前提

本轮执行前已完成：

1. 阅读 `docs/status/summary.md`
2. 阅读 `docs/tasks/t-now.md`
3. 阅读 `docs/tech/v1/implementation-prep-round-27.md`
4. 执行 `make check`

本轮实际验证方式：

- 使用仓库内置最小 HTTP 服务
- 使用仓库内置 worker runner 推进任务状态
- 逐个样例走上传、状态查询、结果页访问、结果接口和下载接口

## 2. 本轮联调样例

本轮按既定清单准备并执行以下 5 类样例：

1. 文本型 `PDF`
2. 常规 `DOCX`
3. 常规 `DOC`
4. 无可审查正文样例
5. 不支持格式样例

另统一走查以下 4 类页面：

1. `reviewing`
2. `completed`
3. `failed`
4. `not_found`

## 3. 逐样例结果

### 3.1 文本型 PDF 样例

结果：

- 上传成功，返回任务 ID
- 上传后状态为 `uploaded`
- worker 推进后状态为 `completed`
- 可读取最终结论和风险列表
- 结果页展示正常
- Markdown 结论与报告下载正常

结论：

- 通过
- 未发现阻塞性交付缺口

### 3.3 常规 DOC 样例

结果：

- worker 推进后状态为 `completed`
- 可读取最终结论和风险列表

结论：

### 3.4 无可审查正文样例

结果：

- 上传成功，返回任务 ID
- 上传后状态为 `uploaded`
- worker 在解析阶段结束后将任务推进为 `failed`
- 当前失败提示为“文件无可审查正文”
- 当前错误码为 `DOCUMENT_NO_REVIEWABLE_TEXT`
- 失败态页面可展示错误码和失败说明
- 结果接口返回 `409 RESULT_FAILED`

结论：

- 通过
- 当前失败阶段、失败分类、失败提示和页面反馈均符合 V1 预期
- 未发现阻塞性交付缺口

### 3.5 不支持格式样例

结果：

- 上传阶段直接被拒绝
- 接口返回 `415`
- 错误码为 `UNSUPPORTED_FILE_TYPE`
- 提示语为“仅支持 PDF 或 Word 文件。”

结论：

- 通过
- 当前属于上传前置校验拒绝，不进入任务创建和结果页链路
- 未发现阻塞性交付缺口

## 5. 发现的问题

本轮未发现新的阻塞性交付缺口。

联调过程中唯一出现的偏差，是联调脚本初版把“无可审查正文”的预期错误码写成了错误名称；经复核，真实系统输出为：

- `DOCUMENT_NO_REVIEWABLE_TEXT`

该结果与当前失败语义一致，属于联调脚本判断口径错误，不属于产品或实现缺陷。

## 6. 本轮结论

本轮正式结论为：

- 通过，可进入验收 / 交付

原因如下：

1. 3 个正常样例均完成“上传 -> 状态推进 -> 结果查看 -> Markdown 下载”闭环
2. 2 个失败样例均在正确阶段给出正确反馈
3. `reviewing`、`completed`、`failed`、`not_found` 四类页面均通过走查
4. 未发现新的阻塞性交付缺口
