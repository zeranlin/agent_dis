## 文档目的

这份文档用于承接“继续进入下一轮解析链路深化”的最新指令，在当前输入侧边界内继续补一个最小实现包，进一步提升解析输出与审查输入的衔接稳定性。

## 2. 本轮目标

本轮只处理两件事：

1. 继续把解析片段类型落到条款对象
2. 继续把章节上下文稳定带入审查输入

## 3. 本轮实现点

### 3.1 条款对象补齐 `chapter_title`

当前 `ClauseRecord` 新增：

- `chapter_title`

这样后续审查输入不必再反查章节对象，就可以直接拿到片段所属章节上下文。

### 3.2 条款对象补齐更明确的 `clause_type`

当前不再统一写死为“未分类条款”，而是根据解析块类型映射为：

- `条款片段`
- `段落片段`

这有助于后续审查阶段区分“明确条款命中”与“段落补充命中”。

### 3.3 审查输入继续保留章节上下文

当前解析写入条款对象时，会把：

- 章节标题
- 定位标签
- 片段类型

一起稳定落库，使后续审查输入消费更顺畅。

## 4. 最小测试

本轮补齐以下验证：

1. 条款对象会带上 `chapter_title`
2. 条款对象会区分 `条款片段` 和 `段落片段`
3. 审查输入装配后仍保留这些上下文字段

## 5. 当前结论

本轮不是进入新的解析主题，而是在既有解析深化主线上继续补一层“输出到输入”的结构衔接。

完成这一轮后：

- 条款对象语义更完整
- 审查输入对章节上下文的消费更直接
- 当前实现仍保持 V1 极简边界
