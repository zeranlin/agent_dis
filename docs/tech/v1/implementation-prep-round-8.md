# V1 实现准备第八轮（首版）

## 文档目的

这份文档用于承接结果页字段与错误场景成果，收口结果页最小实现落点、字段消费方式和页面级错误交互方案。

## 1. 产品追溯

本轮实现直接追溯以下已确认文档：

- `docs/status/summary.md`
- `docs/tasks/t-now.md`
- `docs/tasks/current.md`
- `docs/business/v1/page-and-result-spec.md`
- `docs/tech/v1/api-spec.md`
- `docs/tech/v1/implementation-prep-round-7.md`

若后续发现页面落点与产品页面边界冲突，应先回提总负责人，不直接引入复杂前端体系。

## 2. 本轮目标

本轮只聚焦三件事：

1. 给结果页落一个最小可验证实现
2. 明确结果页如何消费现有接口字段
3. 把接口级错误推进为页面级交互反馈

## 3. 本轮范围

### 3.1 本轮要做

1. 新增结果页最小访问路径
2. 服务端渲染最小 HTML 结果页
3. 覆盖完成态、审核中、失败态三种页面反馈
4. 补齐对应最小测试

### 3.2 本轮不做

- 复杂前端框架接入
- SPA 路由体系
- 富文本样式系统
- 权限控制

## 4. 当前实现设计

### 4.1 最小结果页落点

当前新增最小页面入口：

- `GET /review-tasks/{task_id}/page`

这是一个服务端渲染的最小结果页实现，用于：

- 验证结果页已有明确落点
- 验证结果接口字段已被实际消费
- 验证错误交互不再只停留在接口层

### 4.2 字段消费方式

页面当前按三类状态消费现有数据：

#### 完成态

使用以下字段：

- `summary_title`
- `overall_conclusion`
- `conclusion_markdown`
- `report_markdown`
- `risk_count_summary`
- `top_risks`
- `downloadable_files`

页面展示包括：

- 结果状态标签
- 风险统计卡片
- 重点风险摘要
- 最终结论区
- 审查报告区
- 下载链接区

#### 审核中

使用以下字段：

- `status_message`
- `file_name`

页面当前给出：

- 审核中提示
- 当前文件信息
- 刷新或继续轮询的提示说明

#### 失败态

使用以下字段：

- `status_message`
- `error_code`
- `file_name`

页面当前给出：

- 失败状态提示
- 错误码
- 重新提交或检查链路的建议

### 4.3 页面级错误交互

当前页面侧交互不再只依赖 HTTP 状态码，而是按任务状态给出最小可读反馈：

- `reviewing`：继续等待并刷新
- `failed`：显示错误码和重试建议
- `completed`：展示结果与下载入口

当前说明：

- 这仍是最小结果页，不等同于最终产品页
- 当前目标是“有落点、能展示、可验证”

## 5. 最小测试

本轮至少覆盖以下验证：

1. 完成态结果页可被打开并展示关键区块
2. 审核中结果页能展示等待反馈
3. 失败态结果页能展示错误反馈

## 6. 当前结论

实现准备第八轮的目标不是做完整前端，而是先把结果页落成一个最小、可验证的实现点。

完成这一轮后，当前仓库已经具备：

- 结果页最小访问路径
- 页面对结果字段的实际消费
- 页面级错误交互最小落点

后续可继续进入更具体的页面实现、布局优化或更完整交互设计。
