# V1 实现准备第七轮（首版）

## 文档目的

这份文档用于承接结果查询与下载接口成果，收口结果页接入准备所需的最小字段、关键错误场景处理和 Markdown 模板后续稳定化口径。

## 1. 产品追溯

本轮实现直接追溯以下已确认文档：

- `docs/status/summary.md`
- `docs/tasks/t-now.md`
- `docs/tasks/current.md`
- `docs/business/v1/page-and-result-spec.md`
- `docs/tech/v1/api-spec.md`
- `docs/tech/v1/implementation-prep-round-6.md`

若后续发现结果页接入字段与产品页面边界冲突，应先回提总负责人，不直接扩展前端实现范围。

## 2. 本轮目标

本轮只聚焦三件事：

1. 明确结果页最小读取字段
2. 补齐结果与下载接口的关键错误场景
3. 继续稳定 Markdown 模板结构

## 3. 本轮范围

### 3.1 本轮要做

1. 为结果接口补充结果页实际依赖字段
2. 补齐失败态与无效下载类型处理
3. 明确结果页最小读取方式
4. 补齐对应最小测试

### 3.2 本轮不做

- 结果页前端实现
- 风险排序策略深度优化
- 模板样式美化
- 权限控制

## 4. 当前实现设计

### 4.1 结果页最小读取字段

当前结果接口在原有基础上补充以下字段：

- `conclusion_markdown`
- `risk_count_summary`
- `top_risks`
- `generated_at`

这样结果页最小接入时，可直接使用：

1. `overall_conclusion` 或 `conclusion_markdown` 渲染主结论区
2. `report_markdown` 渲染报告正文区
3. `risk_count_summary` 渲染风险统计区
4. `top_risks` 渲染顶部风险摘要区
5. `downloadable_files` 直接驱动下载按钮

### 4.2 关键错误场景

当前新增或明确收口以下错误场景：

- 结果未就绪：`409 RESULT_NOT_READY`
- 任务已失败：`409 RESULT_FAILED`
- 下载类型不存在：`404 DOWNLOAD_NOT_FOUND`
- 任务不存在：`404 TASK_NOT_FOUND`

当前说明：

- 若任务已经进入失败态，不再笼统返回“结果未就绪”
- 这样结果页可以更清晰区分“继续等待”与“任务失败”

### 4.3 模板后续稳定化

本轮不新增复杂样式，只继续强化结构稳定性：

- 结论文档保留文件概览、结论摘要、风险统计、处理建议
- 报告文档保留文件信息、总体结论、报告说明、风险明细

当前模板目标仍是：

- 字段稳定
- 结构可读
- 便于结果页直接引用或渲染

## 5. 最小测试

本轮至少覆盖以下验证：

1. 结果接口返回结果页所需的关键字段
2. 失败任务访问结果接口返回 `RESULT_FAILED`
3. 失败任务下载结果文件返回 `RESULT_FAILED`
4. 无效下载类型返回 `DOWNLOAD_NOT_FOUND`

## 6. 当前结论

实现准备第七轮的目标不是做结果页本身，而是先把“结果页接什么字段、接口如何报错、模板如何保持稳定”收口清楚。

完成这一轮后，结果页接入所需的最小接口契约已经比上一轮更明确，后续可以继续进入结果页接入实现或更完整错误交互设计。
