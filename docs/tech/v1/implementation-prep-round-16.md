# V1 实现准备第十六轮（首版）

## 文档目的

这份文档用于承接“下一轮解析链路深化”指令，继续在不扩范围的前提下提升解析稳定性、正文提取质量和切分输出对后续审查的衔接质量。

## 1. 产品追溯

本轮深化直接追溯以下已确认文档：

- `docs/status/summary.md`
- `docs/tasks/t-now.md`
- `docs/tasks/current.md`
- `docs/business/v1/parsing-and-segmentation-minimum.md`
- `docs/tech/v1/implementation-prep-round-14.md`
- `docs/tech/v1/implementation-prep-round-15.md`

## 2. 本轮目标

本轮只处理三件事：

1. 继续减少正文中的目录噪声，提升解析稳定性
2. 继续提升 `DOCX` 正文提取质量，补齐表格文本进入正文链路
3. 继续优化切分输出，使其更利于后续审查消费

## 3. 本轮深化点

### 3.1 目录噪声过滤

当前清洗规则新增：

- 过滤孤立“目录”标题
- 过滤带页码尾缀的目录行

这样可以减少目录内容误进入正文，降低后续切分和规则命中的噪声。

### 3.2 DOCX 表格文本提取

当前 `DOCX` 提取在段落之外，继续补齐表格文本提取：

- 遍历表格行列
- 将单元格内容按顺序拼成文本行
- 以顺序文本形式并入正文

这与产品口径里“表格先转文本，不做复杂结构恢复”一致。

### 3.3 切分输出与审查消费衔接

当前继续增强两类输出：

1. 章节文本
2. 条款定位标签

具体做法：

- 章节文本不再只保留标题，而是聚合章节下已切出的正文片段
- 条款 `location_label` 增强为“章节标题 / 条款锚点”

这样可以让后续规则审查、证据定位和结果展示更容易消费当前解析输出。

### 3.4 可审查正文阈值微调

本轮同步将正文长度最小阈值从过于保守的口径调整为更贴近当前 V1 实际输入的范围，避免已经具备章节、条款和正文的短文本被误判为不可审查。

## 4. 最小测试

本轮补齐以下验证：

1. 目录噪声行会被过滤，但正文保留
2. `DOCX` 表格文本会进入正文
3. 章节文本会聚合已切出的正文片段
4. 条款定位标签会带上章节上下文

## 5. 当前结论

本轮不是进入新的解析主题，而是在既有真实解析入口上继续做“更稳定可用”的收口。

完成这一轮后：

- 正文噪声更少
- `DOCX` 表格文本覆盖更完整
- 切分输出与后续审查消费衔接更顺
- 当前实现仍保持 V1 极简边界
