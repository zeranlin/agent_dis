## 2. 本轮双小包边界

本轮只合并同一主题下的 2 个小包，不跨主题混包。

### 2.1 小包 A：结果页消费上下文字段

目标：

- 让结果页真正消费结果接口里已经稳定提供的上下文字段

本包只处理：

- 重点风险展示 `chapter_title`
- 重点风险展示 `clause_type`
- 重点风险展示 `review_reasoning`

本包不处理：

- 结果页整体布局重做
- 新增筛选、折叠或交互编排
- 报告模板扩写

### 2.2 小包 B：结果展示口径统一

目标：

- 让结果接口与报告输出复用同一套结果展示辅助逻辑

本包只处理：

- 统一章节上下文提取逻辑
- 统一片段类型提取逻辑
- 统一风险展示排序逻辑

本包不处理：

- 风险模型扩字段
- 审查规则扩写
- 新增结果对象类型

## 3. 本轮实现点

### 3.1 结果页补齐重点风险上下文

当前结果页的“优先关注的风险”不再只显示标题、风险级别、命中位置和风险说明，而是继续补齐：

- `章节上下文`
- `片段类型`
- `审查说明`

这样结果页与结果接口、Markdown 报告三者的风险说明口径进一步对齐。

### 3.2 结果展示 helper 统一复用

当前新增统一的结果展示辅助模块，用于承接：

- 风险排序
- 章节上下文提取
- 片段类型提取
- 重点风险对象组装

这样后续结果接口和报告输出如果继续细化，可沿同一口径推进，减少重复实现和字段漂移。

### 3.3 报告输出排序与结果接口保持一致

当前审查报告风险明细改为按统一严重度顺序输出：

1. 高风险
2. 中风险
3. 低风险

同级再按创建时间顺序排列。

这样结果接口里的 `top_risks` 与审查报告中的风险阅读顺序保持一致，人工复核时更容易对照。

## 4. 最小测试

本轮补齐以下验证：

1. 结果页完成态会显示“章节上下文”
2. 结果页完成态会显示“片段类型”
3. 结果页完成态会显示“审查说明”
4. 审查报告风险明细会按统一严重度顺序输出
5. 审查报告继续保留章节上下文与片段类型

## 5. 风险与边界

本轮风险可控，原因如下：

- 只调整结果消费和展示，不改解析主链路
- 不改任务状态流转
- 不改规则命中逻辑
- 不扩 V1 产品范围

本轮明确不做：

- OCR
- 多文件
- 复杂表格结构恢复
- 复杂版式恢复
- 结果页复杂交互
