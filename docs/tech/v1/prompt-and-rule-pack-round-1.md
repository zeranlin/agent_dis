# 提示词与规则包联合收口首包回提

## 文档目的

本文件用于记录“提示词与规则包联合收口”首包的实现范围、真实样例验证结果和当前结论。

本轮严格限定在以下主题内：

1. 收口审查提示词中的高价值风险优先级
2. 为重点规则补命中定义、正反例和关注项
3. 让新增规则语义真实进入 `LLM` 审查载荷
4. 基于 `SZCG2025000300-A` 做真实联调复核

本轮不处理：

1. 结果归并轻收口
2. 长条款细切分与重复块降权
3. 页面、交付、工作台或其他扩范围主题

## 输入依据

本轮收口直接追溯以下已确认材料：

1. `docs/comms/inbox-t.md`
2. `docs/tech/v1/sample-quality-review-szcg2025000300-a.md`
3. `assets/review/prompts/review-task-instruction.v1.md`
4. `assets/review/rule-packs/default-rule-pack.v1.json`

其中，样例复盘已经明确本轮上一阶段的主要问题为：

1. 高价值风险漏报
2. 规则标题错位
3. 同条款重复命中

## 本轮实现内容

## 1. 提示词收口

已更新 `assets/review/prompts/review-task-instruction.v1.md`，补充以下约束：

1. 严格参考规则中的命中定义、正反例和典型表述
2. 明确优先关注 `R1`、`R3`、`R5`、`R9`、`R12`
3. 明确对“同品牌”“原厂保修”“厂家认证讲师”“执业医师证”“评分量化不足”“本地注册/办公”等高价值表述保持高敏感度
4. 明确风险标题不得与命中规则语义错挂
5. 明确同一条款命中同一规则只输出一次

本轮目标不是让提示词自行做结果归并，而是先把高价值规则召回与语义对齐拉起来。

## 2. 规则包收口

已更新 `assets/review/rule-packs/default-rule-pack.v1.json`，版本提升为 `1.1.0`。

本轮为高优先级规则补充了结构化语义字段：

1. `priority_hint`
2. `hit_definition`
3. `focus_terms`
4. `positive_examples`
5. `negative_examples`
6. `review_focus`

本轮重点补充的规则如下：

1. `R1 地域限制检查`
2. `R2 所有制/规模歧视检查`
3. `R3 资格条件过高检查`
4. `R5 品牌/型号指向检查`
5. `R9 评分项未量化检查`
6. `R12 关键条款缺失/责任失衡检查`

其中关键收口点包括：

1. 为 `R2` 增加反例，避免把最低报价限制误挂为所有制/规模歧视
2. 为 `R12` 明确“责任失衡”语义，不再只按“关键模块缺失”理解
3. 为 `R3`、`R5`、`R9` 增加真实高价值表述和正反例

## 3. 执行器载荷收口

已更新 `app/review_executor.py`。

本轮不再只向 `LLM` 传递规则的基础字段，而是把整条规则对象透传到批次载荷中，使新增的命中定义、正反例和关注项能够真实参与判断。

同时补充了执行级约束：

1. 严格参考规则对象中的扩展语义字段
2. 优先检查高价值规则
3. 对高价值表述优先判断
4. 同一条款同一规则不重复输出

## 4. 自动化验证补充

已更新 `tests/test_parse_worker.py`。

新增和补强的验证点包括：

1. 规则包加载后能读取扩展语义字段
2. 审查提示词已包含高价值规则优先约束
3. 批次载荷会保留规则扩展字段并下发给执行器

## 真实样例验证

## 样例

1. 样例编号：`SZCG2025000300-A`
2. 文件类型：`DOCX`
3. 验证目标：对比本轮收口前后，高价值规则召回和规则语义对齐是否改善

## 本轮主要结果

本轮优化后，真实结果的整体表现为：

1. `overall_conclusion`：本文件存在 `11` 条高风险问题，建议优先复核资格条件、评分规则或定向限制条款
2. `risk_count_summary.high`：`11`
3. `risk_count_summary.medium`：`8`

## 已确认改善

### 1. `R5` 召回明显改善

本轮已能命中“同品牌”相关高价值表述，例如：

- `第四章 技术要求 / 1.5.2 内窥镜控制单元：提供与影像处理平台同品牌内窥镜主机`

这说明上一轮样例复盘中最关键的品牌/型号定向漏报，已经进入稳定审查视野。

### 2. `R3` 对高门槛资质表述的敏感度提升

本轮已能更积极关注以下表述：

1. 原厂保修
2. 厂家认证讲师
3. 执业医师证

这说明“资格条件过高检查”已经从泛化描述开始向样例中的真实高价值门槛靠近。

### 3. `R1` 对地域相关限制的召回提升

本轮已能识别如“深圳市医疗器械行业同类项目业绩不少于 `2` 个”等地域相关竞争门槛线索，说明规则优先级调整已开始生效。

## 当前残留问题

### 1. 重复命中明显放大

当前最突出的问题是：

1. `R3` 在同一长段评分或资格条款中出现多次近义命中
2. 同一风险域在一长段条款内被拆成多条相似风险
3. 结果条数显著膨胀，影响最终结论可信度

### 2. 偏离表与正文重复区域被同步放大

样例中正文与偏离表存在近似内容时，当前高敏感策略会让重复区域都进入命中，进一步抬高风险数量。

### 3. 本轮未处理结果层收敛问题

这些问题已经明确更接近以下主题：

1. 结果归并轻收口
2. 长条款细切分与重复块降权

由于总负责人已拍板本轮不得跨主题混包，因此本轮不回到实现层处理这些问题，只做正式记录。

## 结论

本轮结论为：`首包通过，但仅在“提示词与规则包联合收口”范围内通过。`

具体判断如下：

1. 高价值规则召回较上一轮有明确改善
2. 规则语义错位已有一定缓解
3. 本轮目标“让高价值规则优先进入真实审查视野”已达成
4. 但重复命中和结果膨胀仍然明显，尚不足以作为最终质量收口结论

因此，本轮正式输出应理解为：

- 第一优先级主题已完成首包验证
- 下一步是否进入第二优先级“结果归并轻收口”，需要由总负责人继续拍板
