# 当前整体推进规划

## 文档目的

本文件用于把当前阶段的唯一主线、下一候选主线和启动条件统一收口，避免产品、规则和实现多线并行。

## 一、当前正式判断

当前正式判断如下：

1. `LLM V1` 的最小闭环已经打通
2. 前 `3` 份真实样例已经完成阶段性收口
3. `SZCG2025000300-A` 的人工差异导向规则补强第一轮首包实现已经完成
4. 但首轮补强尚未在最新真实结果中被验证出来
5. 因此当前不进入第二轮能力补强，也不切入默认规则包升级主线

一句话收口：

- 当前先修通“首轮补强真实生效链路”，再考虑扩规则包。

## 二、当前唯一执行主线

当前唯一执行主线固定为：

- `SZCG2025000300-A` 首轮补强链路核查与受控复跑

### 当前主线只做四件事

1. 核查首轮新增 `3` 组规则是否真的进入候选筛选、送审、结果落库和最终报告主链路
2. 查清 `risk.rule_code` 全空的根因
3. 压制品牌 / 型号旧噪声在报价表说明、合同说明、知识产权条款等位置的误报
4. 基于 `300-A` 重新复跑，并明确回提人工 `19` 条埋点中到底新增覆盖了哪些

### 当前主线明确不做

1. 不进入第二轮能力补强
2. 不启动默认规则包升级实现
3. 不扩页面主题
4. 不扩解析大主题
5. 不开新的大而全规则库

## 三、下一候选主线

当前下一候选主线保留为：

- 默认规则包 `1.1` 小步升级

这条主线方向成立，但当前只保留为候选，不作为立即执行主题。

当前最新拍板进一步明确为：

1. 先不着急进入默认规则包 `1.1` 升级准备
2. 当前优先把真实高价值风险点继续做出来
3. 先让系统在真实文件上把该出的风险点尽量出全
4. 等风险点覆盖达到阶段目标后，再回头做规则包 `1.1` 的整理、定版和升级

### 候选范围

若当前主线顺利收口，下一轮优先考虑以下 `7` 类规则：

1. 废标 / 无效投标条款越权检查
2. 不应评分内容入分检查
3. 付款条件失衡检查
4. 违约责任单方失衡检查
5. 验收标准缺失或不可落地检查
6. 质疑投诉路径缺失检查
7. 中小企业政策适用错误检查

### 启动条件

只有同时满足以下条件，才启动默认规则包 `1.1`：

1. `300-A` 首轮补强已在真实结果中被验证出来
2. `rule_code` 丢失问题已经解决
3. 品牌 / 型号旧噪声已明显压降
4. 当前结果已能对照人工埋点清楚说明“新增覆盖提升了哪些”
5. 当前阶段要求优先补齐的高价值风险点已基本做完
