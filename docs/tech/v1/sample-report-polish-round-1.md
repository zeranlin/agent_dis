# `SZCG2025000300-A` 结果复核与报告口径收口第一轮

## 文档目的

本文件记录围绕真实样例 `SZCG2025000300-A` 当前已跑出的结果，所做的结果复核与报告口径收口第一轮。

本轮只处理结果层问题，不回到：

1. 页面主题
2. 提示词大扩张
3. 复杂策略引擎
4. 新的稳定性主题

## 本轮关注点

本轮只围绕以下问题收口：

1. `8` 组风险的表达是否适合人工复核
2. `R3` 噪声命中是否能在结果层进一步压掉
3. 风险标题、风险级别、规则依据、证据片段、位置和报告是否更自然一致
4. 相近 `R5` 风险组展示是否更顺手

## 本轮实现

本轮仅修改结果层：

- `app/result_presenter.py`
- `app/result_aggregator.py`

没有改动：

- 解析链路
- 候选送审链路
- `LLM` 执行链路
- 页面结构

### 1. 风险组展示优先使用真实条款元数据

此前风险组中的：

- `chapter_title`
- `clause_type`
- `unit_label`

较多依赖从 `review_reasoning` 反推，导致：

1. 业务单元标题可能截断
2. 个别位置字段不够稳
3. 报告展示不够自然

本轮改为优先使用 `ClauseRecord` 中已存在的：

- `chapter_title`
- `clause_type`
- `unit_label`
- `unit_name`
- `location_label`

只有拿不到时，才回退到旧的文本反推逻辑。

### 2. 报告中的业务单元口径更自然

本轮新增了“业务单元显示名”组装逻辑：

- 优先显示 `unit_label`
- 若存在合适的 `unit_name`，则显示为 `业务单元（单元名称）`
- 对超长 `unit_name` 做轻量截断，避免报告正文过长

这使得样例中的展示更接近：

- `商务要求项（检测验证）`
- `商务要求项（免费保修期）`
- `商务要求项（培训方案）`

而不是只显示粗标签或从推理文案里截断出来的长串文字。

### 3. 规则编号改为人工可读口径

此前报告中更容易透出内部规则标识，例如：

- `rule_v1_r12`

本轮统一在结果层转换为人工更适合阅读的规则编号：

- `R12`
- `R5`
- `R3`

这次转换只影响展示，不改变底层存储。

### 4. `R3` 噪声命中做展示级抑制

本轮对以下情况做了极小范围的结果层抑制：

- 当 `R3` 同时已经存在更明确的业务单元命中
- 且另有一条 `R3` 落在 `不确定审查对象`

则：

- 保留前者
- 隐去后者的结果组展示

这样做的理由是：

1. 当前主题是结果复核与报告口径收口
2. 该噪声已经不是前置识别能力问题，而是展示层应该帮人工复核减负的问题
3. 这不会影响关键风险保留

### 5. 总体结论与报告说明文案收口

本轮同步收口了：

1. 总体结论文案
2. 报告说明中的优先关注项
3. 风险组明细中的规则编号展示

使结果更适合人工先扫一眼重点，再逐组回看原文。

## 自动化验证

本轮新增并通过了以下验证：

1. 风险组展示优先消费条款元数据
2. 报告中规则编号按 `R` 编码展示
3. 当存在更明确 `R3` 命中时，`不确定审查对象` 上的 `R3` 噪声会被隐藏
4. 既有结果汇总与 worker 闭环测试保持通过

## 真实样例复核结果

本轮在当前真实配置下重新复跑 `SZCG2025000300-A` 后，结果如下：

- 任务状态：`completed`
- 风险组数量：`7`
- 高风险组数量：`4`
- 中风险组数量：`3`

相比上一轮：

- `8 -> 7`

压降来源：

- `R3` 落在 `不确定审查对象` 的 1 条噪声命中已从最终展示中移除

## 结果复核结论

### 一、`R3` 噪声命中

结论：`已进一步收口`

当前最终展示中已不再保留那条：

- `不确定审查对象 / 供应商认证情况 / 专家打分`

当前保留下来的 `R3` 结果更集中在：

1. `商务要求项（免费保修期）`
2. `商务要求项（培训方案）`
3. `商务要求偏离表中的免费保修期`

这比上一轮更适合人工复核。

### 二、`R5` 相近风险组展示

结论：`当前已更顺手，但仍保留三组展示`

当前三组 `R5`：

1. 同品牌内窥镜主机
2. 同品牌 30 度内窥镜
3. 同品牌导光光缆

仍分别展示，但当前已具备以下改进：

1. 排序集中
2. 规则编号统一
3. 业务单元与位置表述更稳定

本轮判断：

- 当前先不再做更高层合并
- 保持三组展示更有利于人工逐条回看真实条款

### 三、总体结论与报告口径

结论：`已更自然`

当前总体结论更新为：

- `本文件归并后存在 4 组高风险问题和 3 组中风险问题，建议优先复核品牌指向、合同责任失衡和资格门槛偏严条款。`

报告说明中也新增了：

- `当前优先关注：关键条款缺失/责任失衡检查、品牌/型号指向检查、资格条件过高检查。`

这比此前更适合：

1. 先看重点
2. 再逐组回看
3. 最后进入原文证据核对

## 本轮结论

本轮正式结论如下：

1. 当前 `7` 组风险的表达已更适合人工复核
2. `R3` 噪声命中已进一步收口
3. 风险点、规则依据、证据片段、位置和报告已更自然一致
4. 当前实现仍保持 `V1` 极简边界，没有扩页面、流程或复杂策略
5. 当前未发现新的阻塞性交付缺口

## 剩余问题

本轮仍保留以下非阻塞问题：

1. `R5` 三组风险仍然相近，但当前判断保留分组更利于人工逐条核对
2. 个别技术条款名称仍较长，虽然已做截断，但后续还可继续细调
3. 当前总体结论仍属于 `V1` 最小报告口径，不是正式法律意见

## 下一步建议

若继续沿当前主线推进，建议顺序如下：

1. 先把本轮结果复核与报告口径收口作为正式基线
2. 如无新的阻塞性交付缺口，可转入验收视角复核
3. 若仍要继续优化，优先只处理超长业务单元标题和结果摘要文案微调，不再扩新主题
