# V1 实现准备第十五轮（首版）

## 文档目的

这份文档用于承接解析链路首轮实现准备复审意见，只修补主链路正确性的 2 个最小缺口，不展开新的解析优化主题。

## 1. 产品追溯

本轮修正直接追溯以下已确认文档：

- `docs/status/summary.md`
- `docs/tasks/t-now.md`
- `docs/tasks/current.md`
- `docs/business/v1/parsing-and-segmentation-minimum.md`
- `docs/tech/v1/implementation-prep-round-14.md`

## 2. 本轮目标

本轮只处理两件事：

1. 修正多级编号条款被误判为章节的问题
2. 修正无可审查片段文件被错误推进到审查队列的问题

## 3. 修正点

### 3.1 收紧章节判定

当前将章节判定收紧为：

- 中文章节标题
- 中文序号标题
- 单层阿拉伯数字标题

同时将判定顺序调整为：

1. 先判定条款
2. 再判定章节

这样可以避免 `1.1`、`1.2` 这类条款编号先被章节规则吞掉。

### 3.2 增加可审查片段硬校验

当前在解析输出完成后增加硬校验：

- 至少生成一个非 `section` 的可审查片段

若只有章节标题、没有 `clause` 或 `paragraph`，则直接按解析失败处理，不推进到 `review_queued`。

## 4. 最小测试

本轮补齐以下验证：

1. `1.1`、`1.2` 这类多级编号行会落成 `clause`
2. 只有章节标题的文件会在解析阶段失败
3. 失败后不会生成审查队列任务

## 5. 当前结论

本轮不是继续加解析能力，而是先把解析链路主路径纠正到与已收口产品口径一致。

完成这一轮后：

- 章节 / 条款切分关系更符合当前 V1 口径
- 无可审查片段文件会停在解析阶段失败
- 失败阶段不再漂移到审查阶段
