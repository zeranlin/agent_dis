# V1 实现准备第十轮（首版）

## 文档目的

这份文档用于承接结果页信息组织细化成果，继续收口页面主次层级、阅读节奏和状态动作建议。

## 1. 产品追溯

本轮实现直接追溯以下已确认文档：

- `docs/status/summary.md`
- `docs/tasks/t-now.md`
- `docs/tasks/current.md`
- `docs/business/v1/page-and-result-spec.md`
- `docs/tech/v1/implementation-prep-round-9.md`

若后续发现页面细化方向与产品边界冲突，应先回提总负责人，不直接扩展复杂前端能力。

## 2. 本轮目标

本轮只聚焦三件事：

1. 继续优化页面主次层级
2. 明确阅读节奏
3. 细化三类状态的动作建议

## 3. 本轮范围

### 3.1 本轮要做

1. 为完成态补充建议阅读顺序
2. 为完成态快速操作区增加更直接的动作入口
3. 为审核中与失败态补充更顺手的操作建议
4. 补齐对应最小测试

### 3.2 本轮不做

- 复杂视觉风格升级
- 动画系统
- 富文本样式系统
- 权限与登录体系

## 4. 当前实现设计

### 4.1 完成态阅读节奏

当前完成态页面新增“建议阅读顺序”区块，明确引导：

1. 先看风险统计
2. 再看重点风险摘要
3. 最后看最终结论与完整审查报告

其目的是降低结果页第一次阅读的跳转成本。

### 4.2 完成态快速动作

当前快速操作区进一步补充：

- 刷新当前结果页
- 查看结果接口
- 下载结果文件

这样页面既可用于人工查看，也可作为联调和调试入口。

### 4.3 审核中与失败态动作建议

当前两类状态进一步明确：

- `reviewing`：先查看状态接口确认当前阶段，再刷新页面或轮询结果接口
- `failed`：先查看状态接口确认失败原因，再重新提交文件

当前目标是让用户在最小页面里也能明确知道下一步动作。

## 5. 最小测试

本轮至少覆盖以下验证：

1. 完成态页面展示建议阅读顺序
2. 完成态页面展示刷新结果页动作
3. 审核中页面展示查看状态接口动作
4. 失败态页面展示重新查看当前页面动作

## 6. 当前结论

实现准备第十轮的目标不是继续加功能，而是把结果页用起来更顺手。

完成这一轮后，当前结果页已经进一步具备：

- 更明确的阅读顺序
- 更直接的操作入口
- 更清楚的状态动作建议

后续可继续进入布局微调或更细的交互文案优化。
