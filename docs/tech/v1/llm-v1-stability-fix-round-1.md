## 2. 本轮目标

本轮目标不是扩展页面、规则或流程，而是只解决一个阻塞主流程验收的问题：

1. 收缩单次 `LLM` 请求负载
2. 增加最小分批、缩批与失败回退能力
3. 让真实样例审查执行更接近稳定进入 `completed`

## 3. 实现内容

### 3.1 `LLM` 请求负载收缩

- 审查输入载荷改为紧凑 `JSON`，减少无效字符开销
- 默认 `OPENAI_MAX_COMPLETION_TOKENS` 从 `800` 收紧到 `512`
- 新增 `batch_char_budget` 配置，用于约束单批次总字符预算

当前默认字符预算口径为：

- `max(clause_max_chars, min(batch_size * clause_max_chars, 1400))`

### 3.2 审查批次调度收缩

- 新增 `_chunk_clauses_for_review(...)`
- 批次切分同时受 `batch_size` 与 `batch_char_budget` 双重约束
- 目标是避免单批次条款数可控但总文本长度失控

### 3.3 失败拆批与回退

- 新增 `_review_clause_batch(...)`
- 单批次请求失败时，先自动拆半重试
- 若已缩到单条款仍失败，则继续缩短该条款 `clause_max_chars`
- 单条款缩短回退的下限为 `300`

3. 不扩规则包范围
