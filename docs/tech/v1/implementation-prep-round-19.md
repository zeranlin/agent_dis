# V1 实现准备第十九轮（首版）

## 文档目的

这份文档用于承接“继续进入下一轮解析链路深化”的最新指令，在不扩范围的前提下继续补一个最小实现包，提升最小执行闭环的可解释性。

## 1. 产品追溯

本轮实现直接追溯以下已确认文档：

- `docs/status/summary.md`
- `docs/tasks/t-now.md`
- `docs/tasks/current.md`
- `docs/business/v1/parsing-and-segmentation-minimum.md`
- `docs/tech/v1/implementation-prep-round-18.md`

## 2. 本轮目标

本轮只处理两件事：

1. 让审查输入的章节上下文和片段类型继续传递到执行说明
2. 让最小执行闭环中的风险说明和证据说明更可解释

## 3. 本轮实现点

### 3.1 输出模式补齐上下文字段

当前 `ReviewInputAssembler` 的最小输出模式中补齐：

- `chapter_title`
- `clause_type`

这样后续执行器和潜在的输出模式可以明确知道当前片段属于哪一章、属于哪类片段。

### 3.2 风险说明增强上下文表达

当前执行器生成中间风险时，不再只写“条款中出现关键词”，而是明确表达：

- 哪一章
- 哪一类片段
- 在哪个定位标签命中

### 3.3 证据说明增强上下文表达

当前证据说明也同步带上：

- 章节上下文
- 片段类型
- 关键词命中说明

这样后续结果汇总和页面展示在消费中间结果时，可直接看到更完整的解释信息。

## 4. 最小测试

本轮补齐以下验证：

1. 输出模式包含 `chapter_title` 和 `clause_type`
2. 风险说明包含章节上下文和片段类型
3. 证据说明包含章节上下文和片段类型

## 5. 当前结论

本轮不是继续加规则能力，而是让“解析输出 -> 审查输入 -> 中间结果”这条最小执行闭环的解释链条更完整。

完成这一轮后：

- 执行说明更容易追溯到输入结构
- 中间风险和证据对象更易读
- 当前实现仍保持 V1 极简边界
