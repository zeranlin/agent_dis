## 本轮前提

本轮建立在以下已确认事实之上：

1. `app/` 现有文本提取、基础切块、审查对象入口识别、候选送审链路和 `LLM` 收缩执行全部继续保留
2. 本轮不扩页面、不扩多文件、不进入复杂策略引擎
3. 本轮只允许围绕 `BACG2025000066-A` 做同主题内泛化修补

## 本轮实现

本轮只补了一个很窄的实现点：

- 在 `app/review_executor.py` 落库前新增 `finding.rule_code` 轻量纠偏
- 仅处理“模型把明显评分语义风险挂到 `R12`”的场景
- 纠偏条件收口为：
- 当前规则包存在 `R9`
- 当前条款属于明确评分单元，或风险标题本身出现“评分 / 量化 / 自由裁量 / 综合评价”等评分语义
- 不再依赖原文中是否出现零散“分”字，避免把合同类 `R12` 误纠偏到 `R9`

同时新增了 3 条自动化回归用例，覆盖：

1. 明确评分单元下的 `R12 -> R9` 纠偏
2. 合同责任失衡条款保持 `R12`
3. 泛单元标签下但风险标题明确为评分问题时，仍可纠偏到 `R9`
