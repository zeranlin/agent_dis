# V1 实现准备第十七轮（首版）

## 文档目的

这份文档用于承接“DOCX 表格重复提取”复审意见，只修补这一处会影响解析输出正确性的最小缺口。

## 2. 本轮目标

本轮只处理一件事：

1. 修正 `DOCX` 表格文本被重复提取的问题

## 3. 修正点

### 3.1 改为按 `w:body` 直属节点顺序提取

当前 `DOCX` 提取不再先全量扫描 `.//w:p`，再单独遍历表格。

改为：

1. 找到 `w:body`
2. 按直属节点顺序遍历
3. 遇到段落提取段落文本
4. 遇到表格提取表格行文本

这样可以避免表格内段落先被段落提取吞一次，后面表格再重复提取一次。

### 3.2 表格单元格文本仍按顺序文本进入正文

当前表格文本仍保持 V1 既定口径：

- 不恢复复杂结构
- 只转成顺序文本
- 每行按 `单元格1 | 单元格2` 方式进入正文

本轮只修正“重复进入”，不改变表格文本的输出样式。

## 4. 最小测试

本轮补齐以下验证：

1. `DOCX` 表格文本仍能进入正文
2. 同一行表格文本只进入正文一次
3. 当前解析主链路不回退

## 5. 当前结论

本轮不是继续加新解析能力，而是把 `DOCX` 表格提取收口到“稳定只进一次”的状态。

完成这一轮后：

- 表格内容不会重复进入 `raw_text`
- 后续切分、规则命中和风险数不再因表格重复而被放大
- 当前实现仍保持 V1 极简边界
