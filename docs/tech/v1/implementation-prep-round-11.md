# V1 实现准备第十一轮（首版）

## 文档目的

这份文档用于承接结果页布局与动作建议优化成果，继续收口页面文案自然度、辅助提示语和状态说明口径。

## 1. 产品追溯

本轮实现直接追溯以下已确认文档：

- `docs/status/summary.md`
- `docs/tasks/t-now.md`
- `docs/tasks/current.md`
- `docs/business/v1/page-and-result-spec.md`
- `docs/tech/v1/implementation-prep-round-10.md`

若后续发现文案优化方向与产品边界冲突，应先回提总负责人，不直接扩展复杂页面能力。

## 2. 本轮目标

本轮只聚焦三件事：

1. 让页面标题和提示语更自然
2. 让三类状态说明更容易理解
3. 保持现有字段消费和页面结构不变

## 3. 本轮范围

### 3.1 本轮要做

1. 微调页面标题与区块标题
2. 微调完成态查看说明
3. 微调审核中与失败态动作建议
4. 补齐对应最小测试

### 3.2 本轮不做

- 新增页面功能
- 改动结果接口结构
- 引入复杂视觉系统
- 扩展前端框架

## 4. 当前实现设计

### 4.1 完成态文案

当前完成态重点文案调整为：

- 页面标题由“结果页最小实现”收口为“结果查看页”
- 阅读引导由“建议阅读顺序”收口为“建议先这样看”
- 风险区块由“重点风险摘要”收口为“优先关注的风险”
- 辅助提示由“查看建议”收口为“查看提示”

目标是让页面更像真实产品文案，而不是实现占位说明。

### 4.2 审核中文案

当前审核中页面进一步强调：

- 系统仍在处理中
- 先看状态接口确认阶段
- 结果生成后再查看结果接口

### 4.3 失败态文案

当前失败态页面进一步强调：

- 先确认失败原因
- 再决定是否重新提交文件或重新触发任务

## 5. 最小测试

本轮至少覆盖以下验证：

1. 完成态页面展示新的自然语言区块标题
2. 审核中页面展示“系统仍在处理中”提示
3. 失败态页面展示新的动作文案

## 6. 当前结论

实现准备第十一轮的目标不是继续加页面能力，而是把现有结果页的语言收口得更自然。

完成这一轮后，当前结果页已经更接近真实可交付页面的表达方式，同时保持了现有极简边界和实现结构。
