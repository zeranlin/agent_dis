# V1 默认规则包升级与规则来源建议（草案）

## 文档目的

这份文档用于回答两个问题：

1. 当前默认规则包是否应该升级
2. 如果按既有审查框架继续补规则，应该优先去哪里查、先补哪些类

本文档是产品侧的升级建议，不直接替代总负责人拍板结果，也不直接替代技术侧实现方案。

## 1. 当前判断

产品侧判断：**默认规则包应该升级**。

但当前不建议一步扩成“大而全规则库”，而应继续沿用 V1 既有策略：

- 先围绕高价值、全国通用、边界较清楚的规则扩一小步
- 先保证解释稳定、证据可抽取、输出结构统一
- 再决定是否继续向程序规则和政策功能规则扩展

## 2. 为什么需要升级

当前默认规则包只有 `12` 条规则，已经覆盖了资格、技术、评分和合同中的部分核心风险，但与 [government-procurement-review-framework.md](/Users/linzeran/code/2026-zn/agent_dis/docs/business/government-procurement-review-framework.md) 中的完整审查框架相比，仍有四类明显缺口：

1. 合法性规则覆盖不足
2. 程序与公开规则覆盖不足
3. 政策功能规则基本未进入默认规则包
4. 合同与履约风险仍主要由 `R12` 单条规则兜底

## 3. 按一级规则域的主规则来源

按当前框架继续补规则时，建议优先使用以下来源体系。

### 3.1 全国基础法源

这部分适合作为所有一级规则域的上位依据：

- 《中华人民共和国政府采购法》
- 《中华人民共和国政府采购法实施条例》
- 《政府采购货物和服务招标投标管理办法》

这三份文件是补规则时最基础的共同底座。

### 3.2 合法性规则

建议优先查：

- 《中华人民共和国政府采购法》
- 《中华人民共和国政府采购法实施条例》
- 《政府采购货物和服务招标投标管理办法》（财政部令第 `87` 号）
- 财政部和中国政府采购网发布的行政裁决指导性案例

适合从这里补的规则包括：

- 采购方式适用是否合法
- 资格条件是否超出法定边界
- 废标或无效投标条款是否越权
- 不应评分内容是否被纳入评分

### 3.3 公平竞争规则

建议优先查：

- 《中华人民共和国政府采购法》
- 《中华人民共和国政府采购法实施条例》
- 《政府采购货物和服务招标投标管理办法》
- 《关于促进政府采购公平竞争优化营商环境的通知》（财库〔`2019`〕`38` 号）
- 财政部行政裁决指导性案例中关于差别歧视待遇、不合理门槛的案例

适合从这里补的规则包括：

- 地域性限制
- 所有制或规模歧视
- 业绩要求过高或定向
- 技术参数指向性

这部分当前已有一定覆盖，更适合做边界收紧和补例，而不是大扩张。

### 3.4 需求合理性规则

建议优先查：

- 《政府采购需求管理办法》（财库〔`2021`〕`22` 号）
- 财政部关于需求管理的政策解读、培训材料和典型问题说明
- 中国政府采购网及地方财政部门发布的标准文本、采购需求模板

适合从这里补的规则包括：

- 需求是否完整明确
- 需求是否可验证
- 是否明显超配
- 文件内部是否前后不一致

### 3.5 评审可解释性规则

建议优先查：

- 《政府采购货物和服务招标投标管理办法》（财政部令第 `87` 号）
- 中国政府采购网关于评分办法、评审因素和主观分控制的案例与解读
- 地方财政部门标准文本中的评分办法模板

适合从这里补的规则包括：

- 主观评分是否过多
- 分值结构是否失衡
- 是否重复加码
- 评分标准是否不可复核
- 不应评分内容是否被纳入评分

### 3.6 合同与履约风险规则

建议优先查：

- 《中华人民共和国政府采购法》
- 《中华人民共和国政府采购法实施条例》
- 《政府采购需求管理办法》
- 财政部门标准文本中的合同条款模板
- 地方采购文件模板与行政裁决案例

适合从这里补的规则包括：

- 付款条件是否失衡
- 违约责任是否单方失衡
- 验收标准是否缺失或不可落地
- 合同条款是否与文件其他部分冲突

### 3.7 程序与公开规则

建议优先查：

- 《政府采购货物和服务招标投标管理办法》（财政部令第 `87` 号）
- 《政府采购质疑和投诉办法》（财政部令第 `94` 号）
- 中国政府采购网信息公开格式规范、公告公示规范
- 财政部行政裁决指导性案例

适合从这里补的规则包括：

- 程序性矛盾
- 澄清修改是否改变实质条件
- 质疑投诉路径是否完整
- 信息公开要求是否体现不足

### 3.8 政策功能规则

建议优先查：

- 《政府采购促进中小企业发展管理办法》（财库〔`2020`〕`46` 号）
- 财政部关于加大政府采购支持中小企业力度的后续通知
- 节能产品、环境标志产品政府采购相关制度与品目清单
- 进口产品政府采购相关规定
- 省市财政部门的地方政策适用指引

适合从这里补的规则包括：

- 中小企业政策是否正确适用
- 绿色采购和节能环保要求是否缺失
- 进口产品采购规则是否满足前提
- 地方政策规则是否纳入

## 4. 当前默认规则包与审查框架的缺口

按现有框架看，当前默认规则包的覆盖情况大致如下。

### 4.1 已有较好起步

- 公平竞争规则
- 需求合理性规则
- 评审可解释性规则

### 4.2 已有起步但仍偏薄

- 合法性规则
- 合同与履约风险规则
- 程序与公开规则

### 4.3 基本尚未进入默认规则包

- 政策功能规则

## 5. 建议的升级顺序

### 5.1 规则包 `1.1` 建议先补的方向

这一轮优先补“全国通用、价值高、边界相对清楚”的规则：

1. 废标/无效投标条款越权检查
2. 不应评分内容入分检查
3. 付款条件失衡检查
4. 违约责任单方失衡检查
5. 验收标准缺失/不可落地检查
6. 质疑投诉路径缺失检查
7. 中小企业政策适用错误检查

### 5.2 规则包 `1.2` 再考虑补的方向

这一轮再补程序和政策功能的扩展规则：

1. 程序性矛盾检查
2. 澄清修改实质变化检查
3. 信息公开要求体现不足检查
4. 绿色采购/节能环保要求缺失检查
5. 进口产品采购前提缺失检查
6. 地方政策规则遗漏检查

## 6. 当前结论

当前更好的产品方案不是“继续只守着现有 `12` 条规则”，也不是“直接做成完整大规则库”，而是：

1. 先正式启动默认规则包升级主题
2. 先做一轮小步、可解释、全国通用的 `1.1` 升级
3. 再根据真实样例效果决定是否继续扩到程序规则和政策功能规则

## 7. 当前拍板结果

总负责人当前正式拍板如下：

1. 规则包升级方向成立
2. `1.1` 小步升级方向成立
3. 但当前不立即启动该主题
4. 当前只把本文件作为下一候选主线的准备材料

当前不立即启动的原因是：

1. `300-A` 首轮补强尚未在最新真实结果中被验证出来
2. 当前仍需先完成主链路核查、旧噪声压制和受控复跑
3. 在当前主线收口之前，不应再引入新的规则包扩张主题

后续启动条件以 [current-overall-plan.md](../../tech/v1/current-overall-plan.md) 为准。
