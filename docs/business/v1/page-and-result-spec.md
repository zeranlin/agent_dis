# V1 页面与结果对象说明（草案）

## 文档目的

这份文档用于把 V1 的页面边界和核心结果对象进一步说清楚，帮助技术负责人直接开展页面实现、接口设计和结果对象定义。

## 1. 页面范围

V1 的页面交互保持极简，只包含以下三个状态页面：

1. 上传页
2. 审核中状态
3. 结果页

不纳入以下页面：

- 复杂工作台
- 任务管理列表
- 在线规则配置页
- 多角色后台管理页

## 2. 页面说明

### 2.1 上传页

目标：

- 让审核人员提交 1 份待审核招标文件

页面内容建议：

- 页面标题
- 文件上传控件
- 支持格式说明：PDF / Word
- 开始审核按钮

用户动作：

- 选择文件
- 上传文件
- 提交进入审核

页面结果：

- 上传成功后自动进入审核中状态

### 2.2 审核中状态

目标：

- 告诉审核人员系统正在执行自动审查

页面内容建议：

- 当前状态提示：审核中
- 基础文件信息
- 等待说明

用户动作：

- 无复杂操作

页面结果：

- 审核完成后自动进入结果页

### 2.3 结果页

目标：

- 让审核人员在线查看最终结论和审查报告
- 提供下载功能

页面内容建议：

- 最终结论
- 审查报告
- 下载最终结论按钮
- 下载审查报告按钮

扩展显示建议：

- 如需要，可折叠展示风险点、证据片段和规则依据

用户动作：

- 在线查看结果
- 下载结果文件

## 3. 结果对象范围

V1 先定义三类核心结果对象：

1. 上传结果对象
2. 审核状态对象
3. 审查结果对象

## 4. 对象说明

### 4.1 上传结果对象

用途：

- 表示文件是否已成功进入系统

建议字段：

| 字段 | 字段说明 |
| --- | --- |
| `task_id` | 本次审查任务唯一标识 |
| `file_name` | 上传文件名称 |
| `file_type` | 文件类型，如 PDF、Word |
| `status` | 当前状态，初始为已上传 |
| `message` | 上传结果提示信息 |

样例：

```json
{
  "task_id": "task_001",
  "file_name": "招标文件.pdf",
  "file_type": "PDF",
  "status": "uploaded",
  "message": "文件已成功上传，系统即将开始审核。"
}
```

### 4.2 审核状态对象

用途：

- 表示当前任务是否仍在审核中

建议字段：

| 字段 | 字段说明 |
| --- | --- |
| `task_id` | 审查任务唯一标识 |
| `status` | 当前状态，如审核中、已完成、失败 |
| `file_name` | 当前审核文件名称 |
| `started_at` | 审核开始时间 |
| `message` | 当前状态提示信息 |

样例：

```json
{
  "task_id": "task_001",
  "status": "reviewing",
  "file_name": "招标文件.pdf",
  "started_at": "2026-03-28T18:30:00+08:00",
  "message": "系统正在自动审核，请稍候。"
}
```

### 4.3 审查结果对象

用途：

- 承载结果页所需的主查看层内容

建议字段：

| 字段 | 字段说明 |
| --- | --- |
| `task_id` | 审查任务唯一标识 |
| `status` | 当前状态，结果页应为已完成 |
| `file_name` | 文件名称 |
| `summary_title` | 结果摘要标题 |
| `overall_conclusion` | 最终结论 |
| `report_markdown` | 审查报告 Markdown 内容 |
| `downloadable_files` | 可下载文件列表 |

样例：

```json
{
  "task_id": "task_001",
  "status": "completed",
  "file_name": "招标文件.pdf",
  "summary_title": "审查已完成",
  "overall_conclusion": "本文件存在明显风险，建议重点关注资格条件和评分规则。",
  "report_markdown": "# 审查报告\n\n## 总体结论\n\n本文件存在明显风险。\n",
  "downloadable_files": [
    {
      "name": "最终结论.md",
      "type": "markdown"
    },
    {
      "name": "审查报告.md",
      "type": "markdown"
    }
  ]
}
```

## 5. 下载文件建议

V1 建议统一以 Markdown 作为首选下载格式。

原因：

- 轻量
- 易生成
- 易渲染
- 便于后续转换为其他格式

当前下载对象建议为：

- 最终结论 `.md`
- 审查报告 `.md`

## 6. 当前结论

V1 的页面和结果对象先收敛为极简闭环：

1. 上传页
2. 审核中状态
3. 结果页

对应对象先收敛为：

1. 上传结果对象
2. 审核状态对象
3. 审查结果对象

## 7. 后续继续细化方向

后续如继续细化，可按分层准则继续展开：

1. 上传页字段校验
2. 审核中轮询策略
3. 结果页信息区块布局
4. 下载文件内容模板
