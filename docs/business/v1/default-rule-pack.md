# V1 默认规则包 1.0（草案）

## 文档目的

这份文档用于沉淀 V1 阶段可默认内置的审查规则包，作为智能体开展自动初审的基础能力集合。

## 规则包定位

V1 的产品定位是“带有默认规则包的政府采购招标文件审查智能体”，而不是依赖运营人员持续手工配规则的规则运营平台。

因此，这份规则包的目标不是追求完备，而是追求：

- 默认可直接运行
- 规则表达清晰
- 输出结构稳定
- 适合当前 V1 范围

## 适用范围

当前默认规则包仅面向以下范围：

- 文件范围：招标文件正文 + 合同草案
- 采购类型：货物 + 服务
- 采购方式：公开招标

## 使用原则

- 默认全量启用这 12 条规则
- `自动判定` 规则直接进入主报告
- `辅助提示` 规则进入疑似问题区
- 当前版本不假设有专门运营人员在线维护规则参数
- 后续通过规则包版本升级方式迭代，而不是靠即时手工配置

## 规则包特点

这版规则包有三个特点：

1. 先覆盖最典型、最常见、最容易由智能体直接识别的风险
2. 先保证输出稳定，再逐步增加复杂规则
3. 先做规则包版本升级，不依赖复杂运营配置

## 当前结论

V1 阶段不宜等待“完备规则库”之后再开始，而应先从“可运营的默认规则包”开始。

这里的“可运营”不是指需要很多运营人员，而是指：

- 默认可启用
- 默认可解释
- 默认可输出统一结构
- 可以按版本持续升级
