# 政府采购招标文件审查框架与细则清单

## 说明

这份文档把一级审查框架和二级细则清单整理成适合产品需求、规则库设计和审核清单设计的结构。

## 一级审查框架

| 一级规则域 | 审查目标 | 主要审查对象 | 典型输出 |
| --- | --- | --- | --- |
| 合法性规则 | 判断条款是否违反法律法规、强制性标准和政策要求 | 采购方式、资格条件、评分规则、废标条款、合同条款 | 越权设定、违法条款、与上位规则冲突 |
| 公平竞争规则 | 判断是否存在排斥、限制潜在供应商的情形 | 资格条件、技术参数、评分标准、业绩要求、地域要求 | 歧视性条款、指向性条款、竞争边界失衡 |
| 需求合理性规则 | 判断采购需求是否完整、明确、合理、可验证 | 技术规格、服务要求、交付要求、验收指标 | 需求模糊、超配、不可验收、前后不一致 |
| 评审可解释性规则 | 判断评分办法是否客观、量化、可执行、可复核 | 评分项、分值结构、主观分、价格分、商务分 | 主观性过强、重复加码、分值失衡 |
| 合同与履约风险规则 | 判断合同与履约条款是否公平、协调、可落地 | 付款条款、违约责任、质保条款、验收条款 | 风险单方转移、付款失衡、验收缺失 |
| 程序与公开规则 | 判断文件是否为后续招标程序留下程序漏洞 | 公告衔接、澄清修改、开评标规则、质疑投诉条款 | 程序冲突、表述不一致、救济路径缺失 |
| 政策功能规则 | 判断政策性采购要求是否正确纳入文件 | 中小企业条款、绿色采购、进口产品、节能环保 | 应写未写、适用错误、政策条款冲突 |

## 二级细则清单

| 一级规则域 | 二级细则 | 主要风险表现 | 重点检测位置 | 产出建议 |
| --- | --- | --- | --- | --- |
| 合法性规则 | 采购方式适用是否合法 | 采购方式与项目性质、金额、适用条件不匹配 | 项目概况、采购公告、采购方式说明 | 标记适用依据并提示人工复核 |
| 合法性规则 | 资格条件是否超出法定边界 | 增设与项目无关资质、证书、年限、奖项 | 资格条件章节 | 输出越权条件与依据条款 |
| 合法性规则 | 废标或无效投标条款是否越权 | 自行扩张法定废标、无效情形 | 投标无效、废标条款 | 标记越权条款并建议收口 |
| 合法性规则 | 评分项是否纳入不应评分内容 | 将门槛项、法定义务项、合规底线项转为加分项 | 评分办法 | 提示评分逻辑不合法或不当 |
| 公平竞争规则 | 是否存在地域性限制 | 本地注册、本地办公、本地业绩、本地驻场作为门槛 | 资格条件、商务要求 | 标记地域歧视风险 |
| 公平竞争规则 | 是否存在所有制或规模歧视 | 仅限国企、龙头企业、大型企业等不当限制 | 资格条件、评分标准 | 标记差别待遇风险 |
| 公平竞争规则 | 技术参数是否指向特定品牌或型号 | 品牌名、专利名、独特参数组合、唯一描述 | 技术需求章节 | 输出疑似指向性参数组合 |
| 公平竞争规则 | 业绩要求是否过高或定向 | 业绩金额、数量、行业范围明显不合理 | 资格条件、评分办法 | 提示过高门槛或定向风险 |
| 需求合理性规则 | 需求是否完整明确 | 关键性能、服务范围、交付边界表述缺失 | 采购需求、服务要求 | 标记信息缺失点 |
| 需求合理性规则 | 需求是否可验证 | 需求描述抽象，无法映射到验收指标 | 采购需求、验收条款 | 提示补充量化指标 |
| 需求合理性规则 | 是否存在明显超配或不相称要求 | 高配堆砌、与采购目的不相称 | 技术需求、预算说明 | 提示超配风险并建议论证 |
| 需求合理性规则 | 文件内部是否前后不一致 | 需求、交付、验收、合同表述冲突 | 跨章节 | 输出冲突条款对照 |
| 评审可解释性规则 | 主观评分是否过多 | 大量“优良中差”式描述，无量化标准 | 评分办法 | 标记主观裁量过大风险 |
| 评审可解释性规则 | 分值结构是否失衡 | 技术分、价格分、商务分明显失衡 | 评分办法 | 输出分值结构分析 |
| 评审可解释性规则 | 是否重复加码 | 同一能力在资格、符合性、评分中重复要求 | 资格条件、评分办法 | 标记重复要求链路 |
| 评审可解释性规则 | 评分标准是否不可复核 | 打分条件模糊，评委难以统一尺度 | 评分细则 | 提示缺乏可操作标准 |
| 合同与履约风险规则 | 付款条件是否失衡 | 付款周期过长、条件过苛、节点不清 | 合同条款 | 标记付款风险并建议调整 |
| 合同与履约风险规则 | 违约责任是否单方失衡 | 风险主要压给供应商，采购人责任缺失 | 合同条款 | 提示责任不对称问题 |
| 合同与履约风险规则 | 验收标准是否缺失或不可落地 | 验收条件模糊、缺少验收方法或依据 | 验收条款、需求条款 | 提示补充验收机制 |
| 合同与履约风险规则 | 合同条款是否与招标文件其他部分冲突 | 合同与需求、评分、响应要求不一致 | 合同条款、跨章节 | 输出冲突条款比对 |
| 程序与公开规则 | 澄清修改是否可能改变实质条件 | 修改内容实质影响竞争但未充分衔接 | 澄清答疑条款 | 标记程序风险并提示复核 |
| 程序与公开规则 | 文件是否存在程序性矛盾 | 开标时间、提交要求、资格审查方式冲突 | 公告、投标须知、流程条款 | 输出流程冲突点 |
| 程序与公开规则 | 质疑投诉路径是否完整 | 救济对象、时限、路径表达不清 | 质疑投诉条款 | 提示补充救济信息 |
| 程序与公开规则 | 信息公开要求是否体现不足 | 关键公开事项未体现或表述模糊 | 公告衔接、文件说明 | 标记公开不充分风险 |
| 政策功能规则 | 中小企业政策是否正确适用 | 预留份额、价格扣除、声明要求适用错误 | 政策条款、评分办法 | 提示政策适用错误 |
| 政策功能规则 | 绿色采购和节能环保要求是否缺失 | 应写未写、引用不清、执行方式不明 | 政策条款、技术需求 | 标记政策遗漏 |
| 政策功能规则 | 进口产品采购规则是否满足前提 | 应当论证审核但文件未体现 | 采购说明、需求说明 | 提示进口产品合规风险 |
| 政策功能规则 | 地方政策规则是否纳入 | 省市特殊管理要求未体现 | 政策条款、地方模板差异处 | 标记适用区域规则遗漏 |

## 面向产品需求的字段建议

如果将上述框架转为产品规则库，每条细则建议至少配置以下字段：

| 字段 | 说明 |
| --- | --- |
| 规则编号 | 规则的唯一标识 |
| 一级规则域 | 所属大类 |
| 二级细则名称 | 规则名称 |
| 适用文件模块 | 主要命中文件位置 |
| 适用区域 | 全国、省级或市级 |
| 触发模式 | 关键词、结构冲突、逻辑冲突、人工复核等 |
| 风险等级建议 | 高、中、低 |
| 输出文案模板 | 给审核人员看的解释文本 |
| 证据抽取要求 | 需要截取哪些原文 |
| 整改建议模板 | 如何修改或补充 |
| 是否必须人工复核 | 是否允许直接给出强结论 |
